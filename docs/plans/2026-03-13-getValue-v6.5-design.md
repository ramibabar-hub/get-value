# getValue v6.5 — Design Document

**Date:** 2026-03-13
**Status:** Approved
**Approach:** Incremental Layer (A) — build on top of existing code, zero risk to existing models

---

## Context

getValue is a personal global financial workstation (React 19 + Vite 7 + FastAPI + Python).
Current state: all state in `useState` inside `StockDashboard.tsx`, no global store, no CSS variables,
inline styles hardcoded throughout 15+ components.

---

| Question | Answer |
|---|---|
| Grok API key | Live key exists in `.env` — full integration from day one |
| Dark/Light theme | Deferred. CSS custom properties system established globally (foundation only) |
| Valueground layout | B — each model becomes an independent draggable card (canvas-style) |
| AI Insights Layer | A — inline ✨ icon per metric, tooltip on click, lazy Gemini call |

## Deliverable 2 — Grok Sentiment Badge

### Backend
- `backend/services/grok_service.py` ← NEW
  - Calls xAI Grok API using `GROK_API_KEY` from `.env`
  - Returns: `score`, `label` (Bullish/Neutral/Bearish), `reason` (1 sentence), `cached_until`
  - In-memory cache: 15 minutes per ticker
- `backend/main.py` ← ADD endpoint `GET /api/grok/sentiment/{ticker}`

### Frontend
- `frontend/src/components/GrokSentimentBadge.tsx` ← NEW
  - Displayed in Overview tab beneath the price header
  - Badge color: green (Bullish), grey (Neutral), red (Bearish)
  - Tooltip: "Why is it moving?" + `reason` text
  - Loading state: animated pulse skeleton
  - sessionStorage cache key: `gv_grok_{ticker}` — 15 min TTL

### Response shape
```json
{
  "ticker": "AAPL",
  "score": 0.72,
  "label": "Bullish",
  "reason": "Strong iPhone upgrade cycle + AI services narrative",
  "source": "grok",
  "cached_until": "2026-03-13T11:00:00Z"
}
```

### Error handling
- On 4xx/5xx: badge shows "Sentiment unavailable" — no crash, no 502 propagation
- Key missing: badge hidden entirely (no error shown to user)

### Zustand store
```ts
interface LayoutStore {
  cardOrder: string[];       // model display order, e.g. ["CF + IRR", "DDM", ...]
  collapsedCards: string[];  // collapsed card IDs
  setOrder:       (order: string[]) => void;
  toggleCollapse: (id: string)      => void;
  resetOrder:     ()                => void;
}
// persisted via zustand/middleware/persist → localStorage key: "gv_layout_v1"
```

### Default order
```ts
["CF + IRR", "CF + IRR Special", "Normalized PE", "DDM", "Industry Multiple", "Piotroski"]
```

### ValuegroundCard anatomy
```
┌─────────────────────────────────────────┐
│ ⠿  CF + IRR                      [−][⋮] │  ← drag handle, collapse, menu
├─────────────────────────────────────────┤
│                                         │
│   <CfIrrTab />  (unchanged)             │
│                                         │
└─────────────────────────────────────────┘
```
- Drag handle: left side grip icon
- Collapse: animates height to 48px (header only)
- Menu (⋮): "Reset position" option only in v6.5
- Card content: existing tab component passed as `children` — **zero changes to model logic**

### Senior Partner rule
The inner components (`CfIrrTab`, `DDMTab`, etc.) are passed as children.
`ValuegroundCard` is a pure layout wrapper — it cannot alter model output.

## Deliverable 4 — AI Insights Layer ✨

### Files
- `frontend/src/hooks/useInsight.ts` ← NEW
- `frontend/src/components/InsightTooltip.tsx` ← NEW

### Hook contract
```ts
interface UseInsightOptions {
  metric:  string;          // "wacc" | "ev_ebitda" | "revenue_cagr" | ...
  value:   number | string | null;
  ticker:  string;
  context: Record<string, unknown>;  // sector, beta, industry etc.
}

const { insight, isLoading, trigger } = useInsight(options);
// trigger() — called on ✨ icon click, lazy (no auto-fetch)
// insight   — string | null
// isLoading — boolean
```

### Cache
- `Map<string, string>` keyed by `"{ticker}:{metric}"` — module-level singleton
- Cleared on ticker change (StockDashboard `useEffect` cleanup)
- No persistence — each session starts fresh (LLM context may be stale otherwise)

### Placement in v6.5
- Overview: 15 metric cells (existing grid) — each gets ✨ icon
- InsightsTab: WACC section rows
- Expansion to FinancialsTab rows deferred to v6.6

### Backend
- Reuses existing `POST /api/gemini/analyze/{ticker}` endpoint
- Request body includes `metric` and `context` fields
- No new endpoint needed
